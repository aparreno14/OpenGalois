# Rule: `galois_group.QQ.deg4.V4@1`

## 1) Rule id
`galois_group.QQ.deg4.V4@1`

## 2) Claim
Proves a fact of the form:

- `GaloisGroup(f: PolyQQ, G: GroupId)`

## 3) Premises
Exactly five premises:

- `Degree(f, 4)` with the same `f` as in the claim.
- `IrreducibleQQ(f)` with the same `f` as in the claim.
- `DiscSquareQQ(f)` with the same `f` as in the claim.
- `ResolventQQ(R, f, p)` with the same `f` as in the claim.
- `FactorizationMonicQQ(R, factors, unit)` for the same quartic resolvent `R`.

## 4) Evidence
None.

## 5) Theoretical justification (normative notes)

Let `f ∈ Q[x]` be irreducible of degree 4, and let `R` be its quartic cubic resolvent in the sense of
Definition 3.1 of Conrad's paper. This rule fixes the same quartic cubic-resolvent family by requiring
that the third argument `p` in `ResolventQQ(R,f,p)` is exactly the canonical `MPolyQQ` object representing

\[
p = x_1x_2 + x_3x_4.
\]

For a monic quartic

\[
f(X)=X^4+aX^3+bX^2+cX+d,
\]

the corresponding resolvent is

\[
R_3(X)=X^3-bX^2+(ac-4d)X-(a^2d+c^2-4bd).
\]

### 5.1 Why the square-discriminant reducible-resolvent branch leads to `V4`

Conrad's quartic classification theorem states:

- if `disc(f)` is a square in `Q` and the quartic cubic resolvent `R3(X)` is reducible over `Q`,
  then the Galois group is the Klein four group `V`. 

In OpenGalois we certify a stronger form of reducibility: the premise
`FactorizationMonicQQ(R, factors, unit)` is required to show that `R` actually splits completely over `Q`
as a product of three monic linear factors.

This stronger premise is mathematically justified by Corollary 3.8 of the paper:

\[
G_f = V \iff R_3(X) \text{ splits completely over } K,
\]

provided `f` is irreducible and `disc(f)` is a square.

Thus the verifier does not rely on a black-box reading of "reducible". It checks the full splitting
pattern of the cubic resolvent directly.

### 5.2 Why complete splitting of the cubic resolvent is the right condition

The cubic resolvent `R3(X)` has the same discriminant as the quartic `f`:

\[
disc(R_3)=disc(f).
\]

Therefore, when `disc(f)` is a square in `Q`, the cubic resolvent also has square discriminant.
If `R3(X)` is reducible over `Q`, Theorem 2.8 for cubics implies that its splitting field over `Q`
is generated by one root and the square root of its discriminant. Since that discriminant is already a
square in `Q`, reducibility forces complete splitting over `Q`.

This is exactly Conrad's Corollary 3.8, and it is the reason that the full linear factorization of `R`
is the natural glass-box premise for the `V4` branch.

### 5.3 Field picture

The splitting field of the quartic has three distinct quadratic subfields, corresponding to the
three transpositions of the roots encoded by the cubic resolvent. In the Klein-four case, the cubic
resolvent already splits over `Q`, so these three "resolvent values" all lie in the base field.

```text
            Splitting field of f
           /         |         \\
          /          |          \\
      Q(√u1)      Q(√u2)      Q(√u3)
          \\          |          /
           \\         |         /
                     Q
```

This is the characteristic field picture of the Klein four group.

## 6) Verifier algorithm (normative)

1. Check there is a verified premise `Degree(f, 4)` bound to the same `f` as the claim.
2. Check there is a verified premise `IrreducibleQQ(f)` bound to the same `f` as the claim.
3. Check there is a verified premise `DiscSquareQQ(f)` bound to the same `f` as the claim.
4. Check there is a verified premise `ResolventQQ(R, f, p)` whose second argument is the same `f` as in the claim.
5. Check that `p` is exactly the canonical `MPolyQQ` object representing `x1*x2 + x3*x4`.
6. Check there is a verified premise `FactorizationMonicQQ(R, factors, unit)` for that same `R`.
7. Decode `factors` as a `PolyQQList` and `unit` as `RatQQ`.
8. Require `unit = 1`.
9. Require `factors.items` to contain exactly three factor references.
10. For each listed factor:
    - decode it as `PolyQQ`,
    - require it to be degree 1,
    - require it to be monic.
11. Decode `G` canonically as a `GroupId` with `system="smallgroup"`.
12. Require `(G.order, G.index) = (4, 2)`.
13. Accept.

This is a theorem rule: the verifier does not recompute the quartic classification from scratch.
Instead, it checks the exact premises corresponding to the `V` branch in Conrad's quartic theory,
with the stronger and more explicit OpenGalois condition that the cubic resolvent is completely split.

## 7) Failure codes
- `E_PREMISE_MISSING` — a required premise is absent.
- `E_PREMISE_BINDING` — a premise exists but does not bind to the same `f` or the same resolvent `R`.
- `E_BAD_RESOLVENT_FAMILY` — the `ResolventQQ` premise does not use the canonical `p = x1*x2 + x3*x4`.
- `E_BAD_FACTORIZATION` — the factorization does not have the required shape (unit ≠ 1, wrong number of factors, or some factor is not monic linear).
- `E_TYPE` — invalid claim shape or object decoding failure.
- `E_GROUP_MISMATCH` — `G` is not `V4` in the `smallgroup` catalog.
